AutoComplete Fields

The <DuxAutoComplete> component implements drop-down auto-complete fields. They can use single or multi-selection. They can restrict the user to values in a list or allow new values to be added. The following are properties of the AutoComplete component.

Property Default Description
name (required) The name of the form field
items (required) Array of items (see Items, below)
allowMulti false True of the user is allowed to select multiple items in the list
multiSeparators [',', ';'] Array of characters that users type to begin a new item in a multi-select list
allowNewItems false True if the user is allowed to add items not in the supplied list of items
validateItem (none) Callback function to validate items before adding to a multi-select list

Items Array

The items property specifies an array of items the user can choose from as they type. Each item in the array is an object with the following required properties.

Property Description
label The label shown in the list. As the user types, the drop-down list shows items that partially match this label
value The value of the item. It can be same as the label, but must be specified. This value must be unique.

Redux Store

The <DuxAutoComplete> component stores data in the Redux store very similar to the <DuxInput> component. The major difference is with multi-select fields. Instead of a value property, an array of selected items is stored in the selectedItems property.

Validating Multi-Select Items

If a callback is specified in the validateItem property, it is called when a user attempts to add a new item to the multi-select list. It is passed the value in question, the field name, and the form name. It returns an error string if the value is invalid or undefined if valid.

Helper Functions

A few helper functions specific to auto-complete fields are provided. There are getAutoCompleteMultiSelectValues, setAutoCompleteSingleSelectValue, and setAutoCompleteMultiSelectValues. See the Helpers topic for more information.
