Getting Started

If you're the ready, fire, aim type (aren't all programmers?), you should find everything you need on this page to start using DuxForm.

This package consists of three React components: <DuxForm>, <DuxInput>, and <DuxAutoComplete>. DuxForm and DuxInput are thin wrappers around <form> and <input>. All attributes such as method, target, or className are passed along to the underlying HTML tags.

Installation

    npm install duxform

If you are using the <DuxAutoComplete> component, you also need to link to the duxautocomplete.css file in the node_modules/duxform/website/dist/ directory. 

    <link rel="stylesheet" href="duxautocomplete.css"/>
You might be tempted to use the duxautocomplete.css file in your project with webpack and an import statement. This will work as long as you don't need to override any of the CSS in the file.

Redux Store

DuxForm stores form state in the Redux store. It requires the root form object in the store. A reducer is supplied to manage this. 

    import {DuxFormReducer} from 'duxform';

    const appReducer = combineReducers({
        ...  // other reducers
        forms: DuxFormReducer
    });

Browser Requirements

DuxForm doesn't place any special requirements on browser support. It has been tested on the modern evergreen browsers (Chrome and Firefox) as well as Internet Explorer 10 and 11. It definitely won't work on Internet Explorer 8 or earlier.
