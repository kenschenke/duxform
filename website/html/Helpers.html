Helpers

DuxForm comes with several helper functions and actions to interact with the forms, fields, and Redux store.

findFormField

This convenience function locates the HTMLElement object for a given field. undefined is returned if the field could not be found. 

    import {findFormField} from 'duxform';

    findFormField(formName, fieldName);

getAutoCompleteMultiSelectValues

This returns the array of selected items in an auto-complete field. The current application state is passed as the first parameter. 

    import {getAutoCompleteMultiSelectValues} from 'duxform';

    getAutoCompleteMultiSelectValues(state, formName, fieldName);

getFormData

This returns a property from the form Redux store. Use it to to retrieve properties other than valid or error. getFormError, isFormValid, and isFormValidOrPristine are actually just convenience functions that call this function. This function is safe to call at any time during application initialization, even before the form has initialized. The current application state is passed as the first parameter. 

    import {getFormData} from 'duxform';

    getFormData(state, formName, propName, defaultValue);

getFormError

This returns the error message for the form. The current application state is passed as the first parameter. 

    import {getFormError} from 'duxform';

    getFormError(state, formName);

getFormFieldData

This returns a property from the field Redux store. Use it to to retrieve properties other than valid or error. getFormFieldError, isFieldValid, and isFieldValidOrPristine are actually just convenience functions that call this function. This function is safe to call at any time during application initialization, even before the form has initialized. The current application state is passed as the first parameter. 

    import {getFormFieldData} from 'duxform';

    getFormFieldData(state, formName, fieldName, propName, defaultValue);

getFormFieldError

This returns the error message for a given field. An empty string is returned if the form or field have not yet been initialized or not been validated yet. The current application state is passed as the first parameter. 

    import {getFormFieldError} from 'duxform';

    getFormFieldError(state, formName, fieldName);

getFormFieldValue

This returns the value for a given field. The default value is returned if the form or field have not yet been initialized. The current application state is passed as the first parameter. 

    import {getFormFieldValue} from 'duxform';

    getFormFieldValue(state, formName, fieldName, defaultValue);

isFieldValid

This returns a boolean value to indicate if a field is valid. False is returned if the field is not yet initialized. The current application state is passed as the first parameter. 

    import {isFieldValid} from 'duxform';

    isFieldValid(state, formName, fieldName);

isFieldValidOrPristine

This returns a boolean value to indicate if a field is valid or pristine. False is returned if the field is not yet initialized. The current application state is passed as the first parameter. This function is helpful if you want to show an invalid indicator but not until the user has changed the field. It's rather alarming to a user if a blank, pristine form shows red errors on all fields. 

    import {isFieldValidOrPristine} from 'duxform';

    isFieldValidOrPristine(state, formName, fieldName);

isFormValid

This returns a boolean value to indicate if a form is valid. False is returned if the form is not yet initialized. The current application state is passed as the first parameter. 

    import {isFormValid} from 'duxform';

    isFormValid(state, formName);

isFormValidOrPristine

This returns a boolean value to indicate if a form is valid or pristine. False is returned if the form is not yet initialized. The current application state is passed as the first parameter. This function is helpful if you want to show an invalid indicator but not until the user has changed the field. It's rather alarming to a user if a blank, pristine form shows red errors on all fields. 

    import {isFormValidOrPristine} from 'duxform';

    isFormValidOrPristine(state, formName);

clearAutoCompleteSelection

    import {clearAutoCompleteSelection from 'duxform';

    clearAutoCompleteSelection(formName, fieldName);

This action clears selections on a single-select or multi-select auto-complete field.

setAutoCompleteSingleSelectValue

This action sets the selected value in a single-selection auto-complete field. 

    import {setAutoCompleteSingleSelectValue} from 'duxform';

    setAutoCompleteSingleSelectValue(formName, fieldName, value, items, valid = true);

The value must match one of the items in the items array. The items parameter must be the same array passed as the items property to the DuxAutoComplete component.

setAutoCompleteMultiSelectValues

This action sets the selected items in a multi-selection auto-complete field. 

    import {setAutoCompleteMultiSelectValues} from 'duxform';

    setAutoCompleteMultiSelectValues(formName, name, values, items, valid = true);

The values parameter is an array of values that each must match one of the items in the items array. The items parameter must be the same array passed as the items property to the DuxAutoComplete component.

setFormFieldValue

This action sets the value of a field. Do not use this action for auto-complete fields. Use setAutoCompleteSingleSelectValue or setAutoCompleteMultiSelectValues instead. 

    import {setFormFieldValid} from 'duxform';

    dispatch(setFormFieldValue(formName, fieldName, value, valid = true));
