Delayed Validation

Validating the field as the user types can be a little aggressive. Especially if the user is being provided feedback as they type. It can be frustrating for users to be told their email address is invalid after typing just one letter. The idleValidateTimeout and onIdleValidate properties of DuxInput allow processing to occur after the user stops typing. This is the perfect time to validate the field value as a whole. The onIdleValidate function returns an error string if the field is invalid or undefined if valid.

The idleValidateTimeout property specifies the number of milliseconds to delay. As the user continues to type, the timer is reset to 0. The onIdleValidate function is called only after the user does not type anything for the specified number of milliseconds. The function should return undefined if the field is valid or an error message string if invalid.

Properties

Property Default Description
onIdleValidate (none) Callback function that is called after idleValidateTimeout milliseconds have elapsed
idleValidateTimeout 0 The onIdleValidate function is called after this number of milliseconds have elapsed

Example

This example demonstrates validating an email address as the user types. Validation does not occur until the user has not typed anything for at least 2 seconds. Notice that if an invalid email address is typed and indicated as invalid, the error message goes away when the user starts typing again. This is because the DuxInput component does not specify an onValidate callback. Without this callback, the field is set to valid when changes occur.
