Validation

You can supply your own custom validation functions at the field level and the form level. Validation functions are called whenever a field changes value, even as the user is typing. For this reason, they should execute quickly without accessing server resources. See the topic on Asynchronous Validation for techniques you can use to validate against server resources or validation that might not be immediate or resource friendly.

Validation callbacks are specified with the onValidate property to the <DuxForm> and <DuxInput> components. If the field or form is not valid, and error message string is returned. If valid, undefined is returned.

Idle Validation

Validating the field as the user types can be a little aggressive. Especially if the user is being provided feedback as they type. It can be frustrating for users to be told their email address is invalid after typing just one letter. See the section on Delayed Validation for more information and an example.

Field Validation

    const validateAge = (value, fieldName, formName) => {
        if (value < 0 || value > 120) {
            return "Age out of range";
        }
    };

    <DuxInput onValidate={validateAge}/>

Form Validation

    const validateForm = (values, formName) => {
        if (values.country === "USA" && values.phone.length !== 10) {
            return "Phone numbers in the United States must be 10 digits long";
        }
    };

    <DuxForm onValidate={validateForm}>
        ...
    </DuxForm>
